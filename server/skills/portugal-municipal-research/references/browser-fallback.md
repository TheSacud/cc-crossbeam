# Browser Fallback Guidance

Usar browser apenas quando WebSearch/WebFetch não chegam.

## Casos típicos

- menus com navegação JS
- documentos escondidos atrás de “document center”
- balcões online com vários cliques antes de mostrar requisitos
- links quebrados onde a página atual substituiu o documento antigo

## O que fazer

1. Confirmar se a página ou documento ainda existe
2. Ler o mínimo necessário para fechar o gap
3. Registar a ação no finding ou no gap fechado

## O que não fazer

- navegar “porque sim”
- substituir fonte normativa por resumo visual
- fechar gaps sem referência concreta à página visitada
